Extended VS Presentation plugin for Eclipse

Extended VS Presentation plugin is a highly customisable and very useful Eclipse skin ("look and feel").

This skin does not only have all "default" Eclipse presentation features but adds much more useful things like fully customizable opened/closed editor lists, really useful tab area with predictable tab ordering policy, clear and space - saving look, much more customization options like the tab size / padding / border / color / fonts, "on-the-fly" themes switch support without to "reboot" Eclipse etc.

Installation

All what you need to install Extended VS Presentation plugin is to put (do not extract!) the Extended VS Presentation jar to you %Eclipse%/plugin directory and restart Eclipse. Then go to "Window->Preferences->General->Appearance" and change "Current presentation" to "Extended VS Presentation". After "OK" you will need to restart Eclipse once.

You don't need to do anymore to install Extended VS Presentation plugin.

For Eclipse 3.1 Update Manager users: you can use special Eclipse update site for this plugin. Go to "Help -> Software Updates -> Find and Install... -> Search for new features to install -> Next -> New Remote Site..." and use the http://andrei.gmxhome.de/eclipse/ as url.

Note: Extended VS Presentation plugin works on Eclipse 3.1.+ only. It is not possible to use it with older Eclipse releases and it is not recommended to use it with 3.2 Milestones.

Note for the Eclipse 3.2M users: because of the nature of milestone builds, some features could not work as expected or simply do not work at all. I will try to suport latest Eclipse 3.2 milestones, but I cannot guarantee that the plugin will work with all/particular 3.2M builds. E.g. there is currently bug 124564 in Eclipse 3.2M4 which lead to enexpected behavoir by detaching views. Another one is bug 119511 which required a special workaround in plugin.

Usage

Simply use it. Don't forget to check out the <Ctrl + E> shortcut for the fully customizable editor list and try to customize preferences under:

Window->Preferences->General->Appearance->Extended VS Presentation

and

Window->Preferences->General->Appearance->Colors and Fonts-> Extended VS Presentation

Why Extended VS Presentation plugin?

I hate the MRU tab order bug which was introduced with Eclipse 3.0 and is still not fixed in 3.1. This very weird "feature" is a part of default Eclipse installation, so you have no chance to fix it until you use the default presentation. On the other side, there was a "VS Presentation" plugin from Willian Mitsuda, which doesn't have many features from the "default" Eclipse presentation but doesn't also have the MRU :)

My "Extended VS Presentation" plugin is derived from "VS Presentation" plugin from Willian Mitsuda - I have greatly improved and extended it with all missing features from the default Eclipse presentation. Moreover, my plugin offers much more useful features as the default presentation (look here).

Again, many thanks to Willian for his good idea / base implementation.
